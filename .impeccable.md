## Design Context

### Users
Solo power user (the developer) who opens this app each morning to get a complete picture of the day — habits to maintain, tasks to complete, money to manage. Context is desktop-only, keyboard-heavy, used daily in focused work sessions. The job is frictionless daily orchestration: see everything, act on anything, move on fast.

### Brand Personality
**Precise, fast, crafted.** The interface should feel like a well-made tool — not decorative, not generic. Every pixel is intentional. The emotional target is **control, calm confidence, and competence** — the feeling of having your life organized in a system that respects your time and attention.

### Aesthetic Direction
- **Visual tone:** Information-dense, dev-tool aesthetic inspired by Raycast and Warp. Dark-first with light mode planned for future support.
- **Anti-references:** Notion (too much whitespace, content-block sprawl), Todoist (too colorful/playful), generic SaaS dashboards (too many cards with drop shadows and gradients). This should feel handcrafted, not template-generated.
- **Theme:** Dark-only currently (cool charcoal with electric indigo accent). Light mode is on the roadmap — design decisions should not preclude it. Use CSS custom properties for all color references to enable theming.
- **Typography:** Inter, 4 sizes (11/13/15/18px), 2 weights (400/600). Locked.
- **Spacing:** 4px grid. Locked.
- **Motion:** Sub-100ms for hover states, 150ms for modals. No GPU-intensive effects. Respect `prefers-reduced-motion`.

### Design Principles
1. **Density over decoration** — Every pixel earns its place. Prefer showing more information over adding whitespace or ornamentation. If it doesn't help the user act, remove it.
2. **Speed is a feature** — Sub-100ms interactions, instant view switches, no loading spinners. Perceived performance is part of the design language.
3. **Keyboard-first, mouse-compatible** — Design for keyboard navigation and shortcuts first. Mouse/trackpad should work seamlessly but is secondary. Power users shouldn't need to reach for the mouse.
4. **Consistency over cleverness** — Use the same patterns everywhere. Same spacing, same interaction states, same motion curves. Predictability builds trust and muscle memory.
5. **Handcrafted, not generated** — No component libraries, no generic templates. Every component is authored with intent. The app should feel like it was built by someone who cares about the details.

### Design System Reference
All tokens are defined in `src/renderer/shared/styles/globals.css` using Tailwind v4's `@theme {}` block. Components use inline React styles with `var(--token-name)` references — no hardcoded values. See the token definitions there as the single source of truth for colors, typography, spacing, radii, and motion.

### Accessibility Baseline
- WCAG AAA contrast on primary text (12.4:1), AA on secondary (4.8:1)
- 44px minimum touch/click targets
- Full keyboard navigation with visible focus rings (2px solid accent)
- `prefers-reduced-motion` support (all transitions disabled)
- Semantic HTML and ARIA landmarks throughout
- No additional accommodations required beyond current implementation
