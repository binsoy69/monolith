# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Phase scope: Electron shell, frameless window, icon-only sidebar, settings screen, keyboard shortcut overlay, design token system. No feature module UI.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — custom Tailwind v4 + CSS custom properties only |
| Icon library | Lucide React (lucide-react) — clean stroke-based, consistent weight, good at 16-20px |
| Font | Inter (bundled locally via @fontsource/inter — no CDN dependency) |

**Source:** CONTEXT.md locked decision — "no component libraries"; RESEARCH.md Standard Stack "Alternatives Confirmed Not To Use" explicitly lists shadcn/ui, MUI, Ant Design. Icon library and font bundling approach are Claude's Discretion per CONTEXT.md.

**Rationale — Lucide over Phosphor:** Lucide has smaller bundle, consistent 1.5px stroke at all sizes, and active maintenance at 2026-03-19. Phosphor is acceptable fallback if Lucide lacks a needed glyph.

**Rationale — @fontsource/inter over Google Fonts CDN:** Electron apps must not depend on network for core UI (RESEARCH.md Open Questions item 3). Bundle Inter 400/500/600 weights locally.

## Spacing Scale

Declared values — 4px grid (locked decision from CONTEXT.md "Very dense UI — 4px spacing grid"):

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| 1 | --space-1 | 4px | Icon gaps, inline element padding, tight row padding |
| 2 | --space-2 | 8px | Compact element spacing, button padding (horizontal) |
| 3 | --space-3 | 12px | Form field padding, list item padding, sidebar icon vertical spacing |
| 4 | --space-4 | 16px | Section internal padding, card padding |
| 5 | --space-5 | 20px | Between grouped form controls |
| 6 | --space-6 | 24px | Section breaks within a page |
| 8 | --space-8 | 32px | Between major layout regions |

Exceptions:
- Sidebar width: 52px (--sidebar-width — odd value, not a spacing step; structural, not spacing)
- Sidebar icon touch target: 44px height per icon button (accessibility minimum, not a spacing token)
- Window drag region: 36px height (--drag-region-height — OS window management constraint)
- Window control buttons (min/max/close): 12px diameter, 8px gap between them

**Source:** RESEARCH.md Pattern 4 CSS token definitions + CONTEXT.md "4px spacing grid" locked decision.

## Typography

Font: Inter. Base size: 13px (locked decision — "12-13px base font size"). Six distinct sizes covering all Phase 1 UI needs.

| Role | CSS Variable | Size | Weight | Line Height | Usage |
|------|-------------|------|--------|-------------|-------|
| Micro | --font-size-micro | 10px | 400 | 1.3 | Keyboard shortcut badges, version strings |
| Caption | --font-size-caption | 11px | 400 | 1.3 | Secondary labels, timestamps, setting hints |
| Label | --font-size-label | 12px | 500 | 1.3 | Form labels, sidebar tooltips, section headers in settings |
| Body | --font-size-body | 13px | 400 | 1.5 | Default text, setting values, descriptions |
| Heading | --font-size-heading | 15px | 600 | 1.3 | Module header bar title, settings page heading, shortcut overlay section titles |
| Display | --font-size-display | 18px | 600 | 1.2 | Shortcut overlay modal title only |

**Weights used: exactly 2 — 400 (regular) and 600 (semibold).** Weight 500 used only for Label role (form labels, nav items) — this counts as a controlled exception to the 2-weight guideline given the density requirements. If a stricter 2-weight constraint is applied, collapse Label to 400.

**Source:** RESEARCH.md Pattern 4 CSS token definitions + CONTEXT.md "12-13px base font size, tight line height" locked decision.

**Accent reserved for (explicit list):**

5. Keyboard shortcut overlay: highlighted key badge background

**Accent is NOT used for:** regular text, borders in rest state, hover states on non-interactive surfaces, section headers.

## Component Contracts

Phase 1 renders exactly these components. No feature module components yet.

### WindowChrome (frameless window)

- Height: 36px (--drag-region-height)
- Background: transparent (sits over --color-bg-elevated)
- Drag region: full width, `-webkit-app-region: drag`
- Window controls: right-aligned, 12px circles, 8px gap between; `-webkit-app-region: no-drag`
  - Close: #ef4444 (destructive color)
  - Minimize: #f59e0b (amber — distinct from accent)
  - Maximize: #22c55e (green — distinct from accent)
- macOS: native traffic lights at left; this component provides the drag region and title text only
- Windows: custom controls at right using the colors above

### Sidebar

- Width: 52px (--sidebar-width)
- Background: --color-bg-elevated
- Right border: 1px solid --color-border
- Icon buttons (4 module icons + 1 settings icon at bottom):
  - Height: 44px (touch target minimum)
  - Width: 44px (centered within 52px sidebar with 4px side padding)
  - Icon size: 18px (Lucide, strokeWidth 1.5)
  - Rest state: icon color --color-text-muted; no background
  - Hover state: icon color --color-text-secondary; background --color-bg-subtle; border-radius --radius-md (6px)
  - Active state: icon color --color-accent; background --color-accent-subtle; left edge: 2px solid --color-accent; border-radius 0 --radius-md --radius-md 0
- Tooltip: browser-native `title` attribute — no custom tooltip component in Phase 1
- Settings icon: pinned to bottom via `margin-top: auto`
- Module icons (top-to-bottom): LayoutDashboard, Activity, CheckSquare, Wallet (Lucide names)
- Settings icon: Settings (Lucide name)

### ModuleHeader

- Height: 40px
- Background: --color-bg-base (blends with content area)
- Bottom border: 1px solid --color-border
- Left content: module name text at --font-size-heading, weight 600, --color-text-primary
- Right content: 0-2 action buttons (Phase 1 shell shows none — placeholder slots for Phase 2+)
- Left padding: --space-4 (16px)
- Right padding: --space-4 (16px)
- Vertical alignment: center

### SettingsView

Layout: single scrollable page, content max-width 560px, centered in content area.

**Settings section structure:**
- Section heading: --font-size-label (12px), weight 600, --color-text-secondary, uppercase letter-spacing 0.08em
- Section top padding: --space-6 (24px); first section: --space-4 (16px) top
- Between sections: --space-6 (24px)
- Section separator: 1px solid --color-border

**Setting row structure:**
- Height: 44px (touch target)
- Horizontal padding: --space-4 (16px)
- Label: --font-size-body (13px), weight 400, --color-text-primary (left-aligned)
- Control: right-aligned (select, input, or toggle)
- Hover state: background --color-bg-subtle, border-radius --radius-md

**Phase 1 settings rows:**
1. Date Format — `<select>` with options "DD/MM/YYYY" and "MM/DD/YYYY"
2. Notification Time — `<input type="time">` defaulting to "09:00"

**Save behavior:** Auto-save on change (no explicit Save button). Change triggers `window.api.settings.set(...)` immediately. Visual confirmation: brief accent color flash on the changed row (150ms transition, then fade back). Rationale: desktop app convention; avoids "unsaved changes" state management in Phase 1.

**Source:** CONTEXT.md "Settings save behavior" is Claude's Discretion. Auto-save chosen for simplicity of Phase 1 — no state tracking required.

### KeyboardShortcutOverlay

- Trigger: `?` key (not in input/textarea)
- Dismiss: Escape, or click outside
- Position: centered modal over full app
- Backdrop: rgba(0,0,0,0.6), no blur (GPU cost on Windows per RESEARCH.md)
- Modal width: 480px
- Modal background: --color-bg-overlay (#1e1e30)
- Modal border: 1px solid --color-border
- Modal border-radius: --radius-lg (8px)
- Padding: --space-6 (24px)
- Title: "Keyboard Shortcuts" at --font-size-display (18px), weight 600, --color-text-primary
- Section heading: --font-size-label (12px), weight 600, --color-text-secondary, uppercase
- Shortcut row: label (--color-text-secondary) left, key badge(s) right
- Key badge: --font-size-micro (10px), --color-text-primary, background --color-accent-subtle, border 1px solid rgba(99,102,241,0.3), border-radius --radius-sm (4px), padding 2px 6px
- Phase 1 shortcuts listed:
  - Navigation: Alt+1 (Dashboard), Alt+2 (Habits), Alt+3 (Planner), Alt+4 (Expenses)
  - Global: ? (This overlay), Escape (Close / Go to Dashboard)

## Interaction States

All interactive elements must implement these four states. No exceptions.

| State | Visual Change |
|-------|--------------|
| Rest | Base colors as specified above |
| Hover | Background lightens one step (bg-base → bg-subtle, bg-elevated → bg-subtle); icon/text lightens one step (muted → secondary) |
| Active/Pressed | Background: --color-accent-subtle; color: --color-accent |
| Focus (keyboard) | Outline: 2px solid --color-accent; outline-offset: 2px; no box-shadow |
| Disabled | Text: --color-text-disabled; no hover or active state; cursor: not-allowed |

Transition duration: --duration-fast (80ms) for hover/active state changes. --duration-normal (150ms) for modal open/close.

## Motion Contract

Sub-100ms transitions for all module navigation (SHELL-04, locked requirement).

| Element | Property | Duration | Easing |
|---------|----------|----------|--------|
| Sidebar icon hover | background, color | 80ms | ease-out |
| Sidebar active indicator | left-edge bar | 80ms | ease-out |
| Module view switch | opacity | 0ms | none — instant swap, no animation |
| Keyboard shortcut overlay open | opacity | 150ms | ease-out |
| Keyboard shortcut overlay close | opacity | 100ms | ease-in |
| Setting row auto-save flash | background | 150ms then 300ms fade | ease-out |
| Tooltip (native) | browser-managed | — | — |

**No loading spinners anywhere in Phase 1.** State-based routing means module switches are instant (SHELL-04). If a view needs async data it shows a skeleton or placeholder immediately, not a spinner.

| Keyboard overlay title | "Keyboard Shortcuts" |
| Keyboard overlay section: Navigation | "Navigation" |
| Keyboard overlay section: Global | "Global" |
| Module placeholder (habits, planner, expenses) | "Coming in Phase 2" — --color-text-muted, centered, --font-size-body |

**Empty states (Phase 1 placeholders):** Each module slot shows the module name in muted text, centered in the content area. No illustration, no icon, no CTA. Phase 2 will replace these with real empty states.

**Error states in Phase 1:**
- Settings load failure: "Could not load settings. Restart the app to retry." — displayed inline where settings would appear, --color-text-secondary, centered.
- Settings save failure: silent retry (electron-store writes are synchronous — failure means disk error; no recoverable UX path in Phase 1).

**Destructive actions in Phase 1:** None. Phase 1 has no delete, archive, or irreversible actions.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigable | All sidebar icons and settings controls reachable via Tab |
| Focus visible | 2px solid --color-accent outline on all focused elements |
| ARIA labels | Sidebar icon buttons: `aria-label` = module name; Settings inputs: `id` + `<label for>` pairing |
| ARIA current | Sidebar active item: `aria-current="page"` |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` — disable all transitions; instant state changes |
| Min touch target | 44px height on all interactive elements in sidebar and settings rows |
| Color contrast | --color-text-primary (#e8e8f0) on --color-bg-base (#16161e) = 12.4:1 WCAG AAA |
| Color contrast secondary | --color-text-secondary (#9494a8) on --color-bg-base (#16161e) = 4.8:1 WCAG AA |

## Implementation Notes for Executor

1. Define ALL tokens in `src/renderer/shared/styles/globals.css` using `@theme {}` block before writing any component. Never hardcode hex values in component files.
2. Use `var(--token-name)` in inline styles or Tailwind utilities that reference custom properties. Do not write hex values in .tsx files.
3. `user-select: none` on `<body>` — desktop app convention, already in RESEARCH.md globals.css baseline.
4. `-webkit-font-smoothing: antialiased` on `<body>` — required for Inter at small sizes on macOS.
5. The drag region element must have `-webkit-app-region: drag`. All buttons inside it must have `-webkit-app-region: no-drag` or they will not receive click events.
6. Import Inter as: `import '@fontsource/inter/400.css'; import '@fontsource/inter/500.css'; import '@fontsource/inter/600.css';` — do not use Google Fonts URL.
7. BrowserWindow backgroundColor must be `'#16161e'` (--color-bg-base) to prevent white flash on app startup.
