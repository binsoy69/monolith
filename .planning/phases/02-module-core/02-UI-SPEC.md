## Spacing Scale

Declared values (4px grid — no 12px or 20px steps):

| Token | Value | Usage |
|-------|-------|-------|
| --space-1 | 4px | Icon gaps, inline padding, checkbox gaps |
| --space-2 | 8px | Compact element spacing, sidebar item gaps |
| --space-4 | 16px | Default element spacing, card padding, header padding |
| --space-6 | 24px | Section padding, module content padding |

## Color

Source: `globals.css` tokens. Values are locked — do not introduce new colors.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | --color-bg-base | #16161e | Page background, main content area |
| Secondary (30%) | --color-bg-elevated / --color-bg-overlay | #1a1a2e / #1e1e30 | Sidebar, cards, modals, context menus |
| Accent (10%) | --color-accent | #6366f1 | See reserved list below |
| Destructive | --color-destructive | #ef4444 | Archive confirmation, delete confirmation only |

Additional surface tokens (use where specified):
- `--color-bg-subtle` (#22223a): Hover states on cards and rows
- `--color-accent-subtle` (rgba 99,102,241,0.15): Active card background on check-off highlight

Text tokens:
- `--color-text-primary` (#e8e8f0): All primary content text
- `--color-text-secondary` (#9494a8): Secondary metadata (streak labels, dates, wallet names)
- `--color-text-muted` (#5a5a72): Empty state text, placeholder text, disabled labels
- `--color-text-disabled` (#3a3a52): Unscheduled habit names (non-interactive)

Border tokens:
- `--color-border` (rgba 255,255,255,0.07): Card borders, section dividers
- `--color-border-focused` (rgba 99,102,241,0.5): Input focus rings, quick-add field focus

**Accent reserved for:**
1. Checked habit checkbox fill
2. Brief card highlight flash on check-off (--color-accent-subtle background)
3. "Today" date label subtle highlight in planner date navigation
4. Active module indicator in sidebar (already established — inset box-shadow)
5. Expense amount text on the log expense form (to indicate active/focused field)
6. Progress summary accent color ("3" in "3/5 completed" — the completed count only)
7. Input focus border (--color-border-focused)

Accent is NOT used for: general hover states, card backgrounds (unChecked), body text, category color dots (those use the preset palette), or wallet balance numbers.

## Interaction States

### Habit Card States

| State | Visual |
|-------|--------|
| Default (unchecked, scheduled) | bg-base background, border --color-border, full opacity |
| Hover (unchecked) | bg-subtle background (--color-bg-subtle), cursor pointer |
| Checked | Checkbox filled --color-accent, card opacity 0.5, sinks to bottom |
| Check-off animation | 80ms: card bg flashes to --color-accent-subtle then fades out (duration-normal: 150ms total) |
| Uncheck animation | 150ms reverse: opacity restores, card pulses --color-accent-subtle once |
| Unscheduled (today) | opacity 0.35, cursor default, no hover effect, checkbox hidden |
| Edit mode (form open) | Card replaced by HabitForm in same position; rest of list dims to opacity 0.6 |

### Task Row States

| State | Visual |
|-------|--------|
| Default | transparent background, full opacity |
| Hover | bg-subtle background |
| Dragging | slight scale (1.02), elevated shadow (0 4px 12px rgba(0,0,0,0.4)), cursor grab |
| Checked/completed | title text strikethrough, opacity 0.45, sinks below unchecked tasks |
| Edit expanded | inline form slides open below row (150ms ease-out), row above dims slightly |

### Quick-Add Input States

| State | Visual |
|-------|--------|
| Default | bg-elevated background, border --color-border |
| Focused | border --color-border-focused, no glow |
| Filled + pending Enter | no change — submits on Enter |

### Expense Modal States

| State | Visual |
|-------|--------|
| Open | Overlay: rgba(0,0,0,0.6) scrim, modal centered, bg-overlay background, radius-lg |
| Amount input focused | border --color-border-focused |
| Form invalid (submit attempt) | error field border --color-destructive, error message 11px below field |
| Submitting | form opacity 0.7, inputs disabled, no spinner (optimistic — should be instant) |

### Optimistic Update State

All write operations show this pattern:
- Update state immediately (Zustand) before IPC resolves
- If IPC fails: revert state + show error toast
- No loading indicator visible during in-flight state (operations complete in <100ms)

## Destructive Actions

All destructive actions follow the same inline confirmation pattern: a brief prompt replaces the trigger element in-place (no modal). Two buttons: confirm (--color-destructive text) and cancel.

| Action | Trigger | Confirmation Copy | Reversal Effect |
|--------|---------|-------------------|-----------------|
| Archive habit | Right-click context menu | Archive "{name}"? | Hides from today view; reversible via archived view |
| Delete task | Right-click > Delete | Delete this task? | Permanent — no undo |
| Delete expense | Right-click > Delete | Delete this expense? This will reverse the wallet deduction. | Wallet balance restored |
| Delete wallet | Edit wallet > Delete | Can't delete — this wallet has expenses (blocked — no confirm needed) | n/a |
| Delete category | Manage > Delete | Can't delete — this category is in use (blocked — no confirm needed) | n/a |

Destructive confirmation button color: `--color-destructive` (#ef4444) text, transparent background. Cancel button: `--color-text-secondary` text.
