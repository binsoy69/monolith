## Spacing Scale

Declared values (4px grid — no 12px or 20px steps):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gaps, inline padding, kbd key padding horizontal |
| `--space-2` | 8px | Compact element spacing, sidebar nav gap, shortcut row padding |
| `--space-4` | 16px | Default element spacing, section label bottom margin |
| `--space-6` | 24px | Card internal padding, overlay padding, section breaks |
| `--space-8` | 32px | Major section breaks |

Exceptions:
- Sidebar nav buttons: 44px hit target (deliberate touch-target minimum, not from scale)
- Sidebar width: 52px (`--sidebar-width`)
- Drag region height: 36px (`--drag-region-height`)
- Command palette: centered fixed overlay, width 560px (between modal sizes used in overlay: 480px for shortcut overlay — palette is wider to accommodate search input)

Source: `globals.css` `@theme` block (existing)

## Color

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--color-bg-base` | `#16161e` | Main background, module content area, dashboard background |
| Secondary (30%) | `--color-bg-elevated` / `--color-bg-overlay` / `--color-bg-subtle` | `#1a1a2e` / `#1e1e30` / `#22223a` | Dashboard cards (`bg-elevated`), command palette backdrop (`bg-overlay`), card hover states (`bg-subtle`) |
| Accent (10%) | `--color-accent` | `#6366f1` | Reserved for specific elements only (see below) |
| Destructive | `--color-destructive` | `#ef4444` | Overdue task badge only in this phase |

**Accent reserved for:**
- Active sidebar nav indicator (inset left border + `accent-subtle` background)
- Focused element outline ring (2px solid `--color-accent`, 1px offset)
- Progress bar fill on habits summary card
- Streak count highlight text (habits card)
- Keyboard shortcut `<kbd>` background (`--color-accent-subtle`) and border

**Accent NOT used for:**
- Card hover states (use `--color-bg-subtle`)
- Regular text or icons in resting state
- Dashboard card borders (use `--color-border`)

**Warning color for overdue badge:**
- Use `#f59e0b` (amber-400) for "N overdue" badge background / text — distinct from destructive red, conveys urgency without alarm
- Token name: `--color-warning` (introduce in this phase if not present)

**Border:**
- `--color-border`: `rgba(255,255,255,0.07)` — card borders, dividers
- `--color-border-focused`: `rgba(99,102,241,0.5)` — input focus borders

Source: `globals.css` color block (existing), CONTEXT.md D-08 (overdue badge), D-14 (focus ring)

## Component Inventory

New components introduced in this phase:

### DashboardView
- Full-width stacked layout, no sidebar-like subdivision
- Three cards in order: Habits, Tasks, Spending
- Each card: `background: var(--color-bg-elevated)`, `border: 1px solid var(--color-border)`, `border-radius: var(--radius-lg)`, `padding: var(--space-6)`
- Card header row: section title (heading weight 600) left-aligned + optional badge right-aligned
- Clicking anywhere on a card navigates to that module (entire card is interactive, `cursor: pointer`)
- Card hover: `background: var(--color-bg-subtle)`, transition `var(--duration-fast)`
- Cards stacked vertically with `gap: var(--space-4)` between them
- View padding: `var(--space-6)` on all sides
- Date header above the cards: "Friday, March 21" — `font-size: var(--font-size-display)`, `font-weight: 600`, `margin-bottom: var(--space-6)`

**Habits card internals:**
- Progress bar: full-width, height 4px, background `var(--color-bg-subtle)`, fill `var(--color-accent)`, `border-radius: 2px`
- Progress label: "3/5 done" — body size, secondary color, right of the card title
- Streak highlights: up to 2 habits shown as `name + streak count` — name in body/secondary, streak count in body/accent
- Empty state: "No habits scheduled today" in body/muted, no icon

**Tasks card internals:**
- Up to 5 incomplete tasks listed: title only, body size, primary color
- "N remaining" count shown as muted body text below list
- Overdue badge: if overdue count > 0, show pill `"N overdue"` — background `var(--color-warning)` at 15% opacity, text `var(--color-warning)`, font-size small, font-weight 600, padding `var(--space-1) var(--space-2)` (4px 8px), border-radius `var(--radius-sm)`
- Empty state: "No tasks for today" in body/muted

**Spending card internals:**
- Headline: "₱450 today" — display size, weight 600, primary color (or "₱0 today" if empty)
- Category list: up to 3 rows — category name (body/secondary) + amount (body/primary) in a flex row with space-between
- Empty state: "₱0 spent today" in body/muted

### CommandPalette
- Mounted at app level alongside `KeyboardShortcutOverlay`
- Triggered by `Ctrl+K` from anywhere (added to `KeyboardRouter`)
- Fixed overlay: full-screen backdrop `rgba(0,0,0,0.6)`, palette anchored at top 20% of viewport, centered horizontally
- Palette container: width 560px, `background: var(--color-bg-overlay)`, `border: 1px solid var(--color-border)`, `border-radius: var(--radius-lg)`, `padding: var(--space-4)`
- Search input: full-width, `background: var(--color-bg-subtle)`, `border: 1px solid var(--color-border)`, `border-radius: var(--radius-md)`, `padding: var(--space-2) var(--space-4)`, font-size body, placeholder "What do you want to do?"
- Input on focus: `border-color: var(--color-border-focused)`, no outline (border handles focus state)
- Action items below input: `margin-top: var(--space-2)`
- Each item row: `padding: var(--space-2) var(--space-4)`, `border-radius: var(--radius-md)`, heading font-size, primary color
- Active/highlighted item: `background: var(--color-accent-subtle)`, text stays primary
- Items list: "Add task", "Log expense", "Check habit" (in that order)
- Type-to-filter: items that don't match the query are hidden (case-insensitive substring match on label)
- No results state: "No matching actions" in body/muted, centered
- Animation: `fadeIn var(--duration-normal) ease-out` (reuses existing keyframe)

### Updated KeyboardShortcutOverlay
- Existing component updated — no structural change, only SHORTCUTS data expanded
- Three sections: "Navigation", "Module Actions", "Quick-Add"
- Section headings: body size, weight 600, secondary color, `text-transform: uppercase`, `letter-spacing: 0.08em` (existing pattern)

## Focus Ring Contract

All focusable elements in this phase (dashboard cards, command palette items, command palette input, any keyboard-interactive element):

- `outline: 2px solid var(--color-accent)`
- `outline-offset: 1px`
- Applied via `:focus-visible` pseudo-class only (not `:focus`) to avoid showing ring on mouse click
- Dashboard cards: when keyboard-focused show the same ring
- Command palette input: border-color change (described above) is sufficient — no additional outline ring on the input itself

Source: CONTEXT.md D-14
