Planned contract for this plan:

```ts
export type TaskPriority = 0 | 1 | 2 | 3

export interface Task {
  id: string
  title: string
  notes: string | null
  date: string
  completed: boolean
  position: number
  createdAt: string
  priority: TaskPriority
  carriedFromDate: string | null
}
```

```ts
overdueCount = count of incomplete tasks where COALESCE(carried_from_date, date) < today
```

    **3. Upgrade `PlannerRepository` and add `carryForwardToDate`:**

    Update all `SELECT` queries to read:

    - `priority`
    - `carried_from_date as carriedFromDate`

    Update `create()` so inserted tasks always use:

    - `priority = 0`
    - `carried_from_date = NULL`

    Update `update(id, data)` so:

    - `priority` can be written through the existing update path
    - when `data.date` is present, the same update clears `carried_from_date = NULL`

    Add:

    ```ts
    carryForwardToDate(targetDate: string): { movedCount: number }
    ```

    Transaction rules:

    1. select incomplete tasks with `date < targetDate`
    2. order them by `COALESCE(carried_from_date, date) ASC, position ASC, created_at ASC`
    3. select incomplete tasks already on `targetDate`
    4. shift today's existing incomplete tasks down by `movedCount`
    5. move overdue tasks to `targetDate`
    6. set `carried_from_date = COALESCE(carried_from_date, date)` for every moved task
    7. assign moved task positions `0..movedCount-1`

    Completed tasks must never be moved.

    **4. Run carry-forward inside the "today" query paths:**

    In `src/main/ipc/planner.ts`:

    - import `getTodayStr` from `../utils/streaks`
    - when `listForDate(date)` is called and `date === getTodayStr()`, call `repo.carryForwardToDate(date)` before `repo.listForDate(date)`

    In `src/main/ipc/dashboard.ts`:

    - instantiate `PlannerRepository(db)`
    - when `date === getTodayStr()`, call `plannerRepo.carryForwardToDate(date)` before task aggregation
    - replace the overdue query with:

    ```sql
    SELECT COUNT(*) as n
    FROM tasks
    WHERE completed = 0
      AND COALESCE(carried_from_date, date) < ?
    ```

    **5. Update `src/renderer/planner/planner-store.ts` and preload bridge:**

    - `Task` objects in store state must include `priority` and `carriedFromDate`
    - `updateTask` must accept `priority`
    - preload `planner.update` typing must line up with the new `priority` field

    No new planner IPC method is needed; use the existing `planner:update`.

    - priority updates persist as `0 / 1 / 2 / 3`

    Update `tests/dashboard-ipc.test.ts` fixture schema so `tasks` includes `carried_from_date`, then add an assertion proving a carried task still contributes to `overdueCount` after its `date` becomes today.
