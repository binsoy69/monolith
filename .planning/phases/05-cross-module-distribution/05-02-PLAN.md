    **2. Create `src/main/repositories/SearchRepository.ts`:**

    Implement this exact public method:

    ```ts
    query(query: string, limit = 8): SearchResult[]
    ```

    Implementation rules:

    - `query.trim()` is the canonical search string
    - return `[]` immediately when the trimmed query is empty
    - run four targeted SQL queries, each capped at `limit`

    Exact search surfaces:

    - habits: `LOWER(habits.name) LIKE ?`
    - tasks: `LOWER(tasks.title) LIKE ? OR LOWER(COALESCE(tasks.notes, '')) LIKE ?`
    - expenses: `LOWER(COALESCE(expenses.notes, '')) LIKE ? OR LOWER(categories.name) LIKE ? OR LOWER(COALESCE(wallets.name, '')) LIKE ?`
    - daily notes: `LOWER(daily_notes.content) LIKE ?`

    Result shaping rules:

    - habit -> `title = habits.name`, `subtitle = 'Habit'`, `snippet = NULL`, `date = NULL`
    - task -> `title = tasks.title`, `subtitle = 'Task'`, `snippet = tasks.notes`, `date = tasks.date`
    - expense -> `title = COALESCE(NULLIF(expenses.notes, ''), categories.name)`, `subtitle = 'Expense'`, `snippet = categories.name`, `date = expenses.date`
    - daily note -> `title = 'Daily note'`, `subtitle = 'Notes'`, `snippet = a 90-character snippet centered on the first match`, `date = daily_notes.date`

    Ranking rules:

    - exact title prefix match -> score `0`
    - generic title match -> score `1`
    - snippet-only match -> score `2`
    - tie-breaker -> newer `date` first, then `title ASC`

    Return the lowest-score `limit` results after merging the four lists.

    **3. Register `search` IPC and preload bindings:**

    Create `src/main/ipc/search.ts` with:

    ```ts
    ipcMain.handle('search:query', (_, data) => repo.query(data.query, data.limit ?? 8))
    ```

    Register it in `src/main/ipc/index.ts` and expose `window.api.search.query(...)` in `src/preload/index.ts`.
